# Feature Specification: 로컬 전용/전역 스킬 모듈 및 MCP 클라이언트

**Feature Branch**: `003-skill-loader-mcp-client`
**Created**: 2026-04-17
**Status**: Draft
**Input**: User description: "SKILL.md 파일을 파싱하여 스킬의 목적과 가이드를 이해하고, 연결된 타겟 스크립트를 호출해 동작을 수행하는 스킬 로더 엔진과, 외부 서비스와 상호작용하기 위한 MCP 클라이언트를 개발한다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스킬 디스커버리 및 로드 (Priority: P1)

에이전트가 기동되면, 로컬(`.agent/skills/`)과 전역(`~/.agents/skills/`) 디렉토리를 탐색하여 설치된 스킬 목록을 자동으로 발견한다. 각 스킬 디렉토리 내의 `SKILL.md` 파일을 파싱하여 스킬의 이름, 설명, 트리거 조건, 실행 대상 스크립트 경로를 구조화된 데이터로 변환한다. 동일한 이름의 스킬이 로컬과 전역에 모두 존재하면 로컬이 우선한다.

**Why this priority**: 스킬을 찾고 파싱하는 것이 전체 스킬 시스템의 기반이다. 이 단계 없이는 어떤 스킬도 실행할 수 없다.

**Independent Test**: 로컬과 전역 디렉토리에 테스트 스킬을 배치하고, 로더가 올바르게 목록을 반환하며 로컬 우선 규칙이 적용되는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 로컬에 2개, 전역에 3개 스킬이 있을 때, **When** 스킬 디스커버리를 수행하면, **Then** 중복 제거 후 로컬 우선 규칙이 적용된 전체 스킬 목록이 반환된다.
2. **Given** SKILL.md가 없는 디렉토리가 skills 폴더에 있을 때, **When** 디스커버리를 수행하면, **Then** 해당 디렉토리는 무시되고 경고가 기록된다.
3. **Given** 로컬과 전역 디렉토리가 모두 존재하지 않을 때, **When** 디스커버리를 수행하면, **Then** 빈 스킬 목록이 반환되고 에러 없이 진행된다.

---

### User Story 2 - 스킬 실행 (Priority: P2)

사용자 또는 에이전트가 특정 스킬을 호출하면, 시스템은 해당 스킬의 SKILL.md에 명시된 타겟 스크립트를 서브프로세스로 실행하고, 표준 출력의 결과를 반환한다. 스크립트 실행 시 `uv run`을 통한 인라인 의존성 관리를 지원한다.

**Why this priority**: 스킬의 핵심 가치는 실행에 있다. 디스커버리(P1) 후 실행까지 동작해야 MVP가 된다.

**Independent Test**: 테스트용 스킬 스크립트를 작성하고, 스킬 실행 엔진이 이를 호출하여 결과를 올바르게 반환하는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 유효한 스킬이 로드되어 있을 때, **When** 해당 스킬을 이름으로 호출하면, **Then** 타겟 스크립트가 실행되고 stdout 결과가 반환된다.
2. **Given** 스킬의 타겟 스크립트가 존재하지 않을 때, **When** 실행을 시도하면, **Then** 스크립트를 찾을 수 없다는 에러가 반환된다.
3. **Given** 스크립트가 비정상 종료할 때, **When** 실행 결과를 받으면, **Then** stderr 내용과 함께 에러가 반환된다.
4. **Given** 스킬 호출 시 인자가 전달될 때, **When** 스크립트가 실행되면, **Then** 인자가 서브프로세스에 올바르게 전달된다.

---

### User Story 3 - MCP 클라이언트 연결 (Priority: P3)

에이전트가 config.yaml에 정의된 MCP 서버에 연결하여, MCP 프로토콜을 통해 외부 도구와 리소스를 사용할 수 있다. MCP 서버가 제공하는 도구 목록을 조회하고, 특정 도구를 호출하여 결과를 받을 수 있다.

**Why this priority**: MCP는 외부 서비스 통합의 표준 프로토콜이지만, 로컬 스킬 시스템(P1, P2)이 먼저 동작해야 에이전트의 기본 확장성이 보장된다.

**Independent Test**: 테스트용 MCP 서버(mock)를 구동하고, 클라이언트가 연결, 도구 목록 조회, 도구 호출을 올바르게 수행하는지 검증한다.

**Acceptance Scenarios**:

1. **Given** config.yaml에 MCP 서버가 정의되어 있을 때, **When** 에이전트가 기동되면, **Then** MCP 서버에 연결되고 사용 가능한 도구 목록이 조회된다.
2. **Given** MCP 서버에 연결된 상태에서, **When** 특정 도구를 호출하면, **Then** 도구 실행 결과가 반환된다.
3. **Given** MCP 서버가 응답하지 않을 때, **When** 연결을 시도하면, **Then** 타임아웃 에러가 발생하고 에이전트는 MCP 없이 계속 동작한다.

---

### Edge Cases

- SKILL.md에 필수 필드(이름, 스크립트 경로)가 누락되면 해당 스킬을 무시하고 경고를 기록한다.
- 스킬 스크립트가 실행 권한이 없으면 권한 에러를 반환한다.
- MCP 서버가 중간에 연결이 끊기면 해당 MCP 도구를 비활성화하고 경고를 기록한다.
- 동일 이름의 스킬과 MCP 도구가 있으면 스킬이 우선한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 로컬(`.agent/skills/`)과 전역(`~/.agents/skills/`) 디렉토리에서 스킬을 자동 탐색해야 한다.
- **FR-002**: 시스템은 각 스킬 디렉토리의 `SKILL.md` 파일을 파싱하여 이름, 설명, 타겟 스크립트 경로, 트리거 조건을 추출해야 한다.
- **FR-003**: 동일 이름의 스킬이 로컬과 전역에 모두 존재할 경우 로컬을 우선 로드해야 한다.
- **FR-004**: 시스템은 스킬의 타겟 스크립트를 비동기 서브프로세스로 실행하고, stdout을 결과로, stderr를 에러로 반환해야 한다.
- **FR-005**: 시스템은 스킬 스크립트 실행 시 인자 전달을 지원해야 한다.
- **FR-006**: 시스템은 config.yaml에 정의된 MCP 서버에 연결하여 도구 목록 조회 및 도구 호출을 수행해야 한다.
- **FR-007**: 시스템은 MCP 서버 연결 실패 시 에이전트의 나머지 기능에 영향 없이 정상 동작해야 한다.
- **FR-008**: 시스템은 로드된 스킬과 MCP 도구의 통합 목록을 제공해야 한다.

### Key Entities

- **SkillDefinition**: 개별 스킬의 메타데이터. 이름, 설명, 타겟 스크립트 경로, 트리거 조건, 소스 범위(LOCAL/GLOBAL)를 속성으로 가진다.
- **SkillResult**: 스킬 실행 결과. 출력 텍스트, 종료 코드, 에러 메시지를 속성으로 가진다.
- **MCPConnection**: MCP 서버 연결 정보. 서버 이름, 전송 방식(stdio/sse), 명령어, 연결 상태를 속성으로 가진다.
- **ToolDefinition**: 스킬 또는 MCP에서 제공하는 도구의 통합 표현. 이름, 설명, 소스(skill/mcp), 호출 방법을 속성으로 가진다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 로컬과 전역에 배치된 스킬이 기동 시 2초 이내에 전체 디스커버리 및 로드가 완료되어야 한다.
- **SC-002**: 로컬 우선 규칙이 모든 이름 충돌 케이스에서 올바르게 적용되어야 한다.
- **SC-003**: 스킬 실행 요청부터 결과 반환까지 에러 없이 완료되어야 한다.
- **SC-004**: MCP 서버 연결 실패 시에도 에이전트가 에러 없이 기동되어야 한다.
- **SC-005**: 스킬과 MCP 도구가 통합 목록에서 함께 조회 가능해야 한다.

## Assumptions

- SKILL.md는 마크다운 형식이며, 스킬 이름은 `# 제목`에서, 스크립트 경로는 본문 또는 특정 섹션에서 추출한다.
- MCP 클라이언트는 `mcp` Python 패키지를 사용하며, stdio 전송 방식을 기본으로 지원한다.
- 스킬 스크립트는 Python, Bash 등 다양한 언어일 수 있으며, 시스템은 스크립트 언어에 무관하게 서브프로세스로 실행한다.
- 스킬 실행 타임아웃은 config에서 설정 가능하며, 기본값은 60초이다.
